# 실전 프롬프트 모음

연결 후 아래처럼 자연어로 요청하면 됩니다. LLM이 적절한 도구를 골라 호출합니다.

## 둘러보기

- "내 wbsgantt 프로젝트 목록 보여줘"
- "○○ 프로젝트의 WBS 트리를 요약해줘"
- "○○ 프로젝트에서 임계 경로에 있는 작업이 뭐야?"

## 프로젝트 통째 만들기

- "3개월짜리 모바일 앱 출시 프로젝트 WBS를 기획·디자인·개발·QA·출시 단계로 만들어줘. 각 단계에 하위 작업 3~5개씩."
- "아래 요구사항으로 새 프로젝트를 만들어줘: …" (요구사항을 붙여넣기)
- 예시 입력 문서: [sample-project.json](sample-project.json)

## 노드 편집

- "'설계' 단계 아래에 'DB 설계' 작업을 추가해줘"
- "'화면 설계' 작업 이름을 '화면 설계 v2'로 바꾸고 기간을 5일로 잡아줘"
- "'구현' 노드를 '개발' 단계 아래로 옮겨줘"
- "'킥오프'를 마일스톤으로 만들어줘"

## 전제작업(종속성)

- "화면 설계가 끝나야 API 설계를 시작하도록 연결해줘" (FS)
- "프론트엔드와 백엔드를 동시에 시작하되 백엔드는 2일 늦게 시작하게 해줘" (SS +2d)
- "○○ 프로젝트의 전제작업 목록을 보여줘"
- "'설계 → 개발' 종속성을 지워줘"

## 작업 정의서(Dictionary)

- "'요구사항 정의' 작업의 정의·산출물·완료 기준을 채워줘"
- "이 작업 정의서에 산출물로 '와이어프레임', '시안'을 추가해줘"
- "'API 설계'의 완료 기준을 'PM·리드 승인'으로 설정해줘"

## 진척

- "'화면 설계' 작업 진척률을 40%로 업데이트해줘" (말단 작업에 입력, 상위는 자동 집계)

> 팁: LLM이 규칙(마일스톤 기간 0·순환 금지·100% Rule 등)을 어기면 서버가 오류로 알려주고 LLM이 스스로 고쳐 다시 시도합니다. 삭제처럼 되돌리기 어려운 작업은 대상을 한 번 확인한 뒤 진행하세요.
