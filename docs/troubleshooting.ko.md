# 문제 해결

## 연결했는데 인증 오류가 나요 (401)

1. **주소 끝의 `/`** — `https://api.wbsgantt.com/mcp/` 처럼 끝에 슬래시가 있어야 합니다.
2. **토큰 복사** — `bwic_pat_` 로 시작하는 전체 문자열을 그대로 붙여넣었는지 확인.
3. **헤더 형식** — `Authorization: Bearer <토큰>` (Bearer 뒤 공백 1칸).
4. **만료·폐기** — 토큰은 발급 90일 후 만료됩니다. 계정 설정 → API 토큰에서 상태를 확인하고, 만료·폐기됐으면 새로 발급해 다시 등록하세요. (오류 코드: `PAT_EXPIRED` / `PAT_INVALID`)

## 새로 추가된 도구가 안 보여요

LLM 클라이언트는 보통 **새 대화를 시작할 때** 도구 목록을 다시 불러옵니다. 서버 업데이트 후 새 도구(예: 전제작업·정의서 편집)가 안 보이면:

- **새 대화를 시작**하세요. (가장 간단)
- 그래도 안 보이면 커넥터를 **껐다 켜기(재연결)** 하거나 클라이언트를 재시작하세요.
- **토큰 재발급은 필요 없습니다.**

확인: 새 대화에서 *"wbsgantt에서 지금 쓸 수 있는 도구 알려줘"* 라고 물어보면 됩니다.

## 편집은 안 되고 조회만 돼요

토큰이 **읽기 전용(read)** 으로 발급된 경우입니다(오류 코드: `PAT_SCOPE_INSUFFICIENT`). 계정 설정에서 **읽기+쓰기** 권한으로 새 토큰을 발급해 등록하세요.

## 종속성을 연결했는데 오류가 나요

- `DEPENDENCY_CYCLE_DETECTED` — 서로 물고 도는 순환입니다. 방향을 바꾸거나 다른 작업과 연결하세요.
- `DEPENDENCY_ANCESTOR_DESCENDANT` — 부모-자식(조상-자손) 간에는 연결할 수 없습니다.
- `DEPENDENCY_DUPLICATE` — 이미 같은 종속성이 있습니다.

## 일정(날짜)이 바로 안 바뀌어요

전제작업·기간 변경 후 임계 경로(CPM) 재계산은 **비동기**로 처리됩니다. 보통 수 초 내 반영되니 잠시 후 `get_wbs_tree` 로 다시 확인하세요.

## 그 외

해결되지 않으면 [Issues](../../issues) 에 클라이언트 종류·오류 메시지(토큰 값은 제외)를 남겨주세요.
